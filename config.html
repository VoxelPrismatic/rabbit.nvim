Configuration
Rabbit is quite advanced, so please read types carefully.

Types
NvimHlColor: string
This is a color string.
  • #RRGGBB
    Hex color string
  • name
    Neovim named color
  • :HlGroupName
    Samples the color of a highlight group. Eg :Normal will sample the foreground, background, or special color of Normal
  • :HlGroupName#prop
    Samples color prop of the highlight group. Eg :Normal#fg will sample the property of the foreground color (fg) of Normal. This way, you can pass the foreground color of one color as the background to another.
NvimHlKwargs
This mirrors vim.api.keyset.highlight, but has some fancy features for dynamic coloring.
~ key type description

bg NvimHlColor Background color. For the color inheritance, the default property will be bg.
sp NvimHlColor Special color, eg for underlines. For the color inheritance, the default property will be sp.
blend integer Blend factor (opacity) for background color, from 0%-100%
others link string Link/alias to another highlight group
default boolean If set, the existing highlight group will NOT be changed
force boolean If set, all attributes from the existing highlight group will be cleared
styles bold boolean Bold text
italic boolean Italic text
strikethrough boolean Strikethrough text
underline boolean Underline text
undercurl boolean Undercurl text
underdouble boolean Underdouble text
underdotted boolean Underdotted text
underdashed boolean Underdashed text
reverse boolean Reversed text
standout boolean Standout text. Normally displayed as reverse text.
nocombine boolean Do not combine special text decorations (underlines & strikes)
legacy cterm string[] An array of the following styles: bold, italic, strikethrough, underline, undercurl, underdouble, underdotted, underdashed, inverse, reverse, standout, nocombine, altfont. This is a legacy option, and it is recommended you use the above boolean options instead.
ctermfg integer ANSI foreground color, as specified by the user's terminal emulator
ctermbg integer ANSI background color, as specified by the user's terminal emulator

Config
You may set up an LSP, all these types are available.
Rabbit.Config
Parent configuration class
key type description
colors Rabbit.Config.Colors
Rabbit.Config.Colors
In your config, this will be formed as the following. All colors are of type NvimHlColor | NvimHlKwargs. For defaults, only the color is inherited. Bold & italic must be specified here. 
				{
				    [group] = {
				        [key] = value...
				    }
				}
group key default description
types title :Normal Title text
index :Comment Entry index
tail :Comment Right-aligned text with additional details, eg buffer/window numbers
head :NonText Additional details before the main entry label
collection :Constant A collection/folder of entries. Keep in mind that plugins may support changing the color on a per-collection basis.
preview :rabbit.types.plugin Box around previewed buffers. By default, it simply matches the current plugin
plugin :Normal The color of this highlight group changes in accordance with the currently open plugin
files file :Normal File name color, eg init.lua in ~/git/rabbit.nvim/init.lua
path :NonText Path color, eg ~/git/rabbit.nvim/ in ~/git/rabbit.nvim/init.lua
term :Constant The little $ indicating that a buffer is a terminal buffer
void :Function If a buffer has no name, it will be shown as #nil XX, where XX is the buffer number
closed :DiagnosticVirtualTextError When a buffer is closed, it will be highlighted in red
modified :WarningMsg When a buffer is modified, a little dot with this highlight group will appear next to the filename. This exists so you can remove bufferline.nvim from your config
paint rose RosePine or HotPink If you use RosePine, it will use Rose. Otherwise, it will fall back to HotPink.
